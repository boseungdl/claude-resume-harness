# claude-resume-harness

Claude Code(oh-my-claudecode) 위에서 동작하는 **이력서·자기소개서 작성 하네스**입니다.
채용공고(JD) 분석 → 초안 → 퇴고의 3단계 워크플로를, 작성 패스와 검토 패스를 분리한 서브에이전트 구조로 강제합니다.

이 저장소는 **재사용 가능한 하네스 코어만** 담습니다. 실제 이력서 내용·포트폴리오 문서(DOC)는 포함하지 않습니다.

## 구성

```
CLAUDE.md                         # 하네스 규칙(단일 진실 원천·순서 강제·자기승인 금지)
.claude/
├── skills/resume/SKILL.md        # 3단계 워크플로: 분석 → 초안 → 퇴고
└── agents/
    ├── recruiter.md              # 채용전문가 — JD 분석(analyze) / 서류 통과 검증(screen)
    ├── storyteller.md            # 커리어 서사 아크 설계
    ├── copywriter.md             # 진짜 재료를 공감·구체 문장으로 포장
    └── editor.md                 # 문장 퇴고(톤·중복·AI 문체·두괄식·글자수)
```

## 핵심 원칙

- **단일 진실 원천**: 모든 사실(수치·기간·자격·경험·역할)은 지원자 본인이 유지하는 두 파일에서만 인출합니다.
  - `about-me/프로필-원천정보.md` — 커리어·정체성·경력·자격
  - `about-me/프로젝트-원천정보.md` — 프로젝트 사실(역할·스택·성과 라벨·근거)
- **순서 강제**: `01-분석.md`가 없으면 이력서·자소서 초안을 만들지 않습니다.
- **자기승인 금지**: 퇴고는 작성 컨텍스트와 격리된 recruiter(screen)·editor 패스로만 수행합니다.
- **정직성**: repo 근거 없는 성과 수치 창작 금지, `[BLANK]` 항목 사용 금지.

## 사용법

1. 이 저장소의 `CLAUDE.md`와 `.claude/`를 대상 프로젝트 루트에 배치합니다.
2. 대상 프로젝트에 `about-me/프로필-원천정보.md`, `about-me/프로젝트-원천정보.md`를 준비합니다(하네스가 참조하는 사실 원천 — 이 저장소에는 포함되지 않음).
3. Claude Code에서 `/resume` 스킬 또는 "공고 분석해줘 / 이력서 써줘 / 퇴고해줘" 같은 요청으로 실행합니다.
4. 산출물은 `about-me/지원/{회사}-{직무}-{YYYYMM}/`의 `00-공고 → 01-분석 → 02-이력서 → 03-자소서` 순서로 쌓입니다.
